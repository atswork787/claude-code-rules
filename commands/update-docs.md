# ドキュメント更新

## 使用法

`/update-docs`

引数なし。package.json と .env.example を情報源としてドキュメントを同期する。

## 適用条件（実行トリガー）

- package.json の scripts セクションに変更があった場合
- .env.example に変更があった場合
- docs/ 配下のドキュメントに大規模な変更があった場合

## 実行手順

真実の情報源（package.json と .env.example）からドキュメントを同期:

1. package.json の scripts セクションを読む
   - スクリプトリファレンステーブルを生成
   - スクリプト名から推測できる役割の説明を付与（`_comment` フィールド等が存在する場合は内容を含める）

2. .env.example を読む
   - すべての環境変数を抽出
   - 目的・値の例（例: `DATABASE_URL=postgres://localhost:5432/mydb`）・必須/任意の区別を文書化

3. docs/CONTRIB.md を更新（既存ファイルが存在する場合は上書き）:
   - 開発ワークフロー
   - 利用可能なスクリプト
   - 環境セットアップ
   - テスト手順

4. docs/RUNBOOK.md を更新（既存ファイルが存在する場合は上書き）:
   - デプロイ手順
   - 監視とアラート
   - 一般的な問題と修正
   - ロールバック手順

5. 古いドキュメントを特定:
   - docs/ 配下で90日以上変更されていない .md ファイルを検索
   - 手動レビュー用にリスト

6. 今回の更新で変更・新規作成されたファイルとその概要を表示し、.reports/docs-diff.txt に保存

**エラーケース:**

- `.env.example` が存在しない場合はステップ2をスキップし、その旨を報告する
- `docs/` ディレクトリが存在しない場合は自動作成する
- `.reports/` ディレクトリが存在しない場合は自動作成する
