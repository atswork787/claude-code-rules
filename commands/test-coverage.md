# テストカバレッジ

テストカバレッジを分析し、欠落しているテストを生成:

1. カバレッジ付きでテストを実行: npm test --coverage または pnpm test --coverage

2. カバレッジレポートを分析（coverage/coverage-summary.json）

3. 80%カバレッジしきい値を下回るファイルを特定

4. カバレッジが不足している各ファイルについて:
   - テストされていないコードパスを分析
   - 関数のユニットテストを生成
   - APIの統合テストを生成
   - 重要なフローのE2Eテストを生成

5. 新しいテストが合格することを確認

6. 前後のカバレッジメトリクスを表示

7. プロジェクトが80%以上の全体カバレッジに達することを保証

焦点:

- ハッピーパスシナリオ
- エラー処理
- エッジケース（null、undefined、空）
- 境界条件
