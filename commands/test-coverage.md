# テストカバレッジコマンド

このコマンドは、カバレッジ分析と不足テストの生成を行うために **tdd-guide** エージェントを呼び出します。

## このコマンドが行うこと

1. **カバレッジを計測** - テストを実行してカバレッジレポートを生成
2. **不足箇所を特定** - しきい値を下回るファイルを洗い出す
3. **テストを生成** - 不足しているユニット・統合テストを作成
4. **テストを実行** - 生成したテストが合格することを確認
5. **結果を報告** - 改善前後のカバレッジメトリクスを表示

## 動作方法

tdd-guide エージェントは以下を行います:

1. **カバレッジ付きでテストを実行**（使用するパッケージマネージャーに合わせて選択）:

   ```bash
   npm test --coverage
   pnpm test --coverage
   yarn test --coverage
   ```

2. **カバレッジレポートを分析**（Jest: `coverage/coverage-summary.json`、または設定ファイルで指定された出力先）

3. **ファイル単位でしきい値を下回るファイルを特定**（ファイル個別の基準）:
   - 通常コード: ファイルカバレッジ 80%未満
   - 財務計算・認証・セキュリティ・コアビジネスロジック: ファイルカバレッジ 100%未満

4. **カバレッジが不足している各ファイルについて**:
   - テストされていないコードパスを分析
   - 関数のユニットテストを生成（ハッピーパス・エラー処理・エッジケース・境界条件）
   - APIエンドポイント・データベース操作の統合テストを生成（外部HTTP/DBはモックせず実サービスで検証すること）
   - 重要なユーザーフロー（ログイン・決済等）のE2Eテストは `/e2e` コマンドを使用
   - エッジケース（null、undefined、空、境界値）はユニットテストで対応し、E2Eでは扱わない

5. **生成したテストを実行**して合格を確認

6. **改善前後のカバレッジを表示**:

   ```text
   ファイル                  | 変更前 | 変更後 | 差分
   -------------------------|--------|--------|-----
   src/api/auth.ts          |  62%   |  85%   | +23%
   src/lib/calculator.ts    |  45%   | 100%   | +55%
   全体                     |  71%   |  82%   | +11%
   ```

7. **全体カバレッジが80%以上**に達することを保証（ファイル単位の改善を積み上げた結果として全体80%以上を目標とする）

## 使用する場合

以下の場合に `/test-coverage` を使用:

- 既存コードのカバレッジを底上げしたいとき
- PR前にカバレッジ基準を満たしているか確認したいとき
- `/tdd` で実装後にカバレッジ全体を検証したいとき

## 他のコマンドとの統合

- まず `/plan` で構築内容を設計
- 新規実装には `/tdd` を使用（テスト先行で開発）
- E2Eテストの追加は `/e2e` を使用
- ビルドエラーが発生した場合は `/build-fix` を使用
- 実装全体をレビューするには `/code-review` を使用
