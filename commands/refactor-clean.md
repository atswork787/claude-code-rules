# リファクタークリーン

テスト検証を伴う安全なデッドコード識別と削除。

## 適用対象

TypeScript / JavaScript プロジェクト。

## 前提条件

以下のツールがインストール済みであること（未導入の場合は `npx` 経由で実行）:

- knip
- depcheck

## 除外対象（分析・削除対象から外すこと）

- `dist/`, `build/`, `.next/` 等のビルド成果物
- `*.config.ts` / `*.config.js`（設定ファイル）
- `index.ts` / `index.js`（バレルファイルは誤検知が多いため）

---

## 手順

1. デッドコード分析ツールを実行:
   - knip: 未使用のエクスポート、ファイル、依存関係を包括的に検索
   - depcheck: knip で検出できない依存関係の漏れを補完

2. `.reports/dead-code-analysis.md` に以下を含むレポートを生成:
   - ツールごとの検出結果（ファイルパス、対象シンボル名）
   - 重要度分類（SAFE / CAUTION / DANGER）
   - 削除候補の一覧と削除理由

3. 発見事項を重要度で分類:
   - SAFE: テストファイル、未使用のユーティリティ
   - CAUTION: APIルート、コンポーネント
   - DANGER: 設定ファイル、メインエントリーポイント

4. SAFE 分類の項目のみを削除対象として提案する。
   CAUTION / DANGER は削除せず、レポートに記録するのみとする。

5. 各削除前に:
   - プロジェクトのテストコマンド（`package.json` の `test` スクリプト）を実行し、終了コード 0 であることを確認する
   - 変更を適用する
   - テストを再実行し、終了コード 0 であることを確認する
   - テストが失敗した場合: `git restore <削除したファイルパス>` でロールバックし、その項目を削除候補から除外して次の項目へ進む

6. 以下の内容でサマリーを表示:
   - 削除したファイル数 / シンボル数
   - スキップした項目（CAUTION / DANGER / テスト失敗）の件数
   - 削除できなかった項目と理由

---

まずテストを実行せずにコードを削除しないこと！
