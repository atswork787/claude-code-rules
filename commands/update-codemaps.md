# コードマップ更新

## 適用条件（実行トリガー）

- ソースファイルの新規作成または削除
- ソースファイル内の import 文の追加・削除・変更
- ソースファイルの移動またはリネーム

コードベース構造を分析し、アーキテクチャドキュメントを更新:

1. インポート、エクスポート、依存関係について src/ 配下の .ts / .tsx / .js / .jsx ファイルをスキャン（node_modules/, dist/, .git/ は除外）
2. 以下の形式で簡潔なコードマップを生成（1ファイルあたり最大100行以内、実装詳細は省略し、シグネチャ・依存関係のみ記載）:
   - codemaps/architecture.md - 全体アーキテクチャ
   - codemaps/backend.md - バックエンド構造
   - codemaps/frontend.md - フロントエンド構造
   - codemaps/data.md - データモデルとスキーマ

   各コードマップには以下を含める:
   - モジュール一覧（パスと役割）
   - 主要な依存関係グラフ
   - エクスポートされる関数・クラスの一覧（シグネチャのみ）
   - 最終更新タイムスタンプ（ISO 8601形式）

3. 直前のコミット時点のファイルとの差分パーセンテージを計算
4. 変更行数が全コードマップの総行数に対して30%を超える場合、更新前にユーザー承認を要求
5. 各コードマップに新鮮さタイムスタンプを追加
6. レポートを .reports/codemap-diff.txt に保存

codemaps/ および .reports/ ディレクトリが存在しない場合は自動作成する。

分析にはTypeScript/Node.jsを使用（環境が存在しない場合はシェルスクリプトで代替可）。実装の詳細ではなく、高レベルの構造に焦点を当てる。
