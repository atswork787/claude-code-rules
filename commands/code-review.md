# コードレビュー

コミット前の変更（ステージング済み・未ステージング含む）の包括的なセキュリティと品質レビュー:

1. 変更されたファイルを取得: `git diff --name-only HEAD`
1. 各ファイルの変更内容を取得: `git diff HEAD -- <ファイルパス>`
1. 各変更されたファイルについて、以下をチェック:

**セキュリティ問題（クリティカル）:**

- ハードコードされた認証情報、APIキー、トークン
- SQLインジェクションの脆弱性
- XSSの脆弱性
- 入力検証の欠落
- 安全でない依存関係
- パストラバーサルのリスク

**コード品質（高）:**

- 50行を超える関数
- 800行を超えるファイル
- 4レベルを超えるネストの深さ
- エラー処理の欠落
- 変更差分内のconsole.logステートメント（全体スキャンは /verify を使用）
- TODO/FIXMEコメント
- パブリックAPIのドキュメントの欠落（TypeScriptの場合は型定義、JSの場合はJSDoc）

**ベストプラクティス（中）:**

- ミューテーションパターン（代わりにイミュータブルを使用）
- コード/コメントでの絵文字使用
- 新しいコードのテストの欠落
- アクセシビリティの問題（a11y）

**スタイル・軽微（低）:**

- 命名規則の不一致
- 不要なコメント・コードの重複
- マジックナンバーの使用

1. 以下の形式でレポートをチャットに出力:

```markdown
## コードレビュー結果
### [ファイルパス:行番号]
- 深刻度: [クリティカル/高/中/低]
- 問題: [説明]
- 修正案: [推奨される修正]
```

1. クリティカルまたは高の問題が見つかった場合:
   - 処理を停止し、発見された問題の一覧を出力する
   - ユーザーに「問題が解決されるまでコミットを行わないこと」と明示的に警告する
   - 自動的なgit操作は実行しない

セキュリティ脆弱性のあるコードを承認しないでください！

## 引数（省略可）

- `staged` - ステージング済みの変更のみ（`git diff --cached`）
- `all`    - HEADからの全変更（デフォルト）
