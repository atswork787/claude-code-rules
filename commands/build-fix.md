# ビルドと修正

TypeScriptとビルドエラーを段階的に修正（エラー数が少ない場合向け。
多数のエラーが想定される場合は `build-error-resolver` エージェントを使用）:

1. ビルドを実行（プロジェクトのパッケージマネージャーに合わせて選択）:
   - npm run build
   - pnpm build
   - yarn build
   - bun run build
   - ビルド成功後は型チェック（`npx tsc --noEmit`）も実行
   - Lintエラーがビルドに影響する場合: `npx eslint .` も確認

2. エラー出力を解析:
   - ファイルごとにグループ化
   - 重要度でソート（クリティカル: ビルド全体をブロック、高: 単一ファイル失敗、中: 警告）

3. 各エラーについて:
   - エラーコンテキストを表示（前後5行）
   - 問題を説明
   - 修正を提案
   - 修正を適用
   - ビルドを再実行
   - エラーが解決されたことを確認（`npx tsc --noEmit` を再実行してエラー件数が減少したことを確認）

4. 停止する場合（ユーザーに状況を報告し、修正待ちで処理を中断）:
   - 修正が新しいエラーを導入した → 該当の変更を元に戻す
   - 同じエラーが3回の試行後も持続 → エラー詳細を提示してガイダンスを求める
   - ユーザーが一時停止を要求

5. サマリーを表示:

   ```text
   修正: X件 / 残り: Y件 / 新規エラー: Z件
   ビルドステータス: PASS / FAIL
   ```

安全のため一度に1つのエラーを修正！
