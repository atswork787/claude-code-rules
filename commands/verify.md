# 検証コマンド

現在のコードベースの状態に対して包括的な検証を実行します。

## 指示

この正確な順序で検証を実行。ビルドチェック（Step 1）が失敗した場合は停止し、それ以外のステップは失敗しても処理を継続して最後に全エラーをまとめて報告する:

1. **ビルドチェック**
   - `package.json` の `scripts.build` を実行。存在しない場合は `CLAUDE.md` またはプロジェクト設定から特定する
   - 失敗した場合、エラーを報告して停止

2. **型チェック**
   - TypeScript/型チェッカーを実行（`package.json` の `scripts.typecheck` または `tsc --noEmit`）
   - `ファイルパス:行番号` とともにすべてのエラーを報告
   - 失敗しても次のステップへ継続

3. **Lintチェック**
   - リンターを実行（`package.json` の `scripts.lint`）
   - 警告とエラーを報告
   - 失敗しても次のステップへ継続

4. **テストスイート**
   - すべてのテストを実行
   - 合格/失敗のカウントを報告
   - カバレッジのパーセンテージを報告
   - 失敗しても次のステップへ継続

5. **秘密情報スキャン**
   - `src/` 配下のソースファイルを対象に、ハードコードされたAPIキー・トークン・パスワード・JWTを検索
   - 発見した場所（ファイルパス:行番号）を報告

6. **Console.log監査**
   - `src/` 配下のソースファイルを対象に `console.log` を検索（`node_modules/` および `test/` は除外）
   - 場所（ファイルパス:行番号）を報告

7. **Gitステータス**
   - コミットされていない変更を表示
   - 最後のコミット以降に変更されたファイルを表示

## 出力

簡潔な検証レポートを作成:

```
検証: [PASS/FAIL]

ビルド:    [OK/FAIL]
型:        [OK/Xエラー]
Lint:      [OK/X問題]
テスト:    [X/Y合格、Z%カバレッジ]
秘密情報:  [OK/X発見]
ログ:      [OK/X console.logs]

PR準備完了: [YES/NO]
（全項目がOKの場合YES。ビルド・型・テストのいずれかがFAILの場合NO。Lintの警告のみはYES可）
```

FAILまたはERRORの場合、修正の提案とともにリストします（警告のみの場合は任意）。

## 引数

`$ARGUMENTS` には以下を指定可能:

| モード | 実行ステップ |
| ------ | ----------- |
| `quick` | Step 1（ビルド）+ Step 2（型）のみ |
| `pre-commit` | Step 1（ビルド）+ Step 2（型）+ Step 3（Lint）+ Step 6（Console.log監査） |
| `full` | Step 1〜7（デフォルト） |
| `pre-pr` | Step 1〜7 + 追加のセキュリティスキャン（依存パッケージの脆弱性チェックを含む） |

省略時のデフォルトは `full`。スキップされたステップは出力フォーマットで `[SKIP]` と表示する。
