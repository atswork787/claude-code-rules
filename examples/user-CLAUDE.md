## 核となる哲学

複雑なタスクにはTaskツールで特化エージェントを使用してください。

**主要原則:**
1. **エージェント第一**: 単独処理よりエージェント委任を優先
2. **並列実行**: 可能な場合は複数のエージェントでTaskツールを使用
3. **実行前の計画**: 複雑な操作にはPlan Modeを使用
4. **テスト駆動**: 実装前にテストを書く
5. **セキュリティ第一**: セキュリティで決して妥協しない

## 個人設定

### プライバシー
- ログ出力時に秘密情報（APIキー/トークン/パスワード/JWT）をマスクすること
- コード出力やログを外部共有する前に機密データを除去すること

### コードスタイル
- コード、コメント、またはドキュメントに絵文字なし
- イミュータブルなアプローチを優先すること。やむを得ない場合は理由をコメントに残すこと
- ファイルは小さく分割すること

### Git
- 慣例的なコミット: `feat:`、`fix:`、`refactor:`、`docs:`、`test:`
- コミット前に常にローカルでテスト
- 小さく集中したコミット
