# プロジェクト固有設定

> 注: 本リポジトリはテンプレートとして管理している。実運用では `.claude/` 直下に配置し、プロジェクトに不要な記述は削除して利用すること。

個人共通ルール（日本語回答、エージェント哲学、コードスタイル等）は
グローバル設定 `~/.claude/CLAUDE.md` で定義済み。ここでは重複せず、
本プロジェクト固有の補足事項のみ記載する。

## ルールファイル

`.claude/rules/` に詳細ルールを配置:

| ファイル | 内容 |
| --------------- | ---------------------------------------------------- |
| coding-style.md | イミュータビリティ、ファイル構成、命名規則、エラー処理 |
| coding-style-typescript.md | TypeScript固有の型安全性、禁止事項、命名規則 |
| testing.md | TDDワークフロー、カバレッジ要件、テストパターン、検証レポート |
| agents.md | エージェントオーケストレーション、優先順位、並列実行 |
| security.md | セキュリティチェック、秘密情報管理、入力検証 |
| patterns.md | APIレスポンス形式、カスタムフック、リポジトリパターン |
| git-workflow.md | コミット形式、PRワークフロー、ブランチ戦略 |
| hooks.md | フックシステム（PreToolUse, PostToolUse等） |
| performance.md | モデル選択戦略、コンテキスト管理、推論レベル |

## プロジェクト構成の参照先

- プロジェクト全体のルール、ファイル構造、パターン: `/CLAUDE.md`
- コーディング規約の詳細: `.claude/rules/coding-style.md`
- テスト・検証の詳細: `.claude/rules/testing.md`
- エージェント運用の詳細: `.claude/rules/agents.md`
