# Everything Claude Code への貢献

貢献してくださってありがとうございます。このリポジトリは、Claude Code ユーザー向けのコミュニティリソースを目指しています。

## 求めているもの

### エージェント

特定のタスクをうまく処理する新しいエージェント:
- 言語固有のレビュワー（Python、Go、Rust）
- フレームワークエキスパート（Django、Rails、Laravel、Spring）
- DevOps スペシャリスト（Kubernetes、Terraform、CI/CD）
- ドメインエキスパート（ML パイプライン、データエンジニアリング、モバイル）

### スキル

ワークフロー定義とドメイン知識:
- 言語のベストプラクティス
- フレームワークパターン
- テスト戦略
- アーキテクチャガイド
- ドメイン固有の知識

### コマンド

便利なワークフローを呼び出すスラッシュコマンド:
- デプロイコマンド
- テストコマンド
- ドキュメントコマンド
- コード生成コマンド

### フック

便利な自動化:
- リンティング/フォーマッティングフック
- セキュリティチェック
- 検証フック
- 通知フック

### ルール

常に従うべきガイドライン:
- セキュリティルール
- コードスタイルルール
- テスト要件
- 命名規則

### MCP設定

新規または改善されたMCPサーバー設定:
- データベース統合
- クラウドプロバイダーMCP
- 監視ツール
- コミュニケーションツール

## 貢献方法

### 1. リポジトリをフォーク

```bash
git clone https://github.com/YOUR_USERNAME/everything-claude-code.git
cd everything-claude-code
```

### 2. ブランチを作成

```bash
git checkout -b add-python-reviewer
```

### 3. 貢献を追加

適切なディレクトリにファイルを配置:
- 新しいエージェント用に `agents/`
- スキル用に `skills/`（単一の.mdまたはディレクトリ可）
- スラッシュコマンド用に `commands/`
- ルールファイル用に `rules/`
- フック設定用に `hooks/`
- MCPサーバー設定用に `mcp-configs/`

### 4. 形式に従う

**エージェント**にはフロントマターが必要:

```markdown
---
name: agent-name
description: 何をするか
tools: Read, Grep, Glob, Bash
model: sonnet
---

ここに指示...
```

**スキル**は明確で実行可能であるべき:

```markdown
# スキル名

## 使用する場合

...

## 動作方法

...

## 例

...
```

**コマンド**は何をするか説明すべき:

```markdown
---
description: コマンドの簡単な説明
---

# コマンド名

詳細な指示...
```

**フック**には説明を含めるべき:

```json
{
  "matcher": "...",
  "hooks": [...],
  "description": "このフックが何をするか"
}
```

### 5. 貢献をテスト

提出前にClaude Codeで設定が動作することを確認してください。

### 6. PRを提出

```bash
git add .
git commit -m "Python コードレビュワーエージェントを追加"
git push origin add-python-reviewer
```

次に、以下を含むPRを開く:
- 何を追加したか
- なぜ有用か
- どのようにテストしたか

## ガイドライン

### すべきこと

- 設定を焦点を絞ってモジュール化する
- 明確な説明を含める
- 提出前にテストする
- 既存のパターンに従う
- 依存関係を文書化する

### してはいけないこと

- 機密データを含める（APIキー、トークン、パス）
- 過度に複雑またはニッチな設定を追加
- テストされていない設定を提出
- 重複する機能を作成
- 代替なしで特定の有料サービスが必要な設定を追加

---

## ファイル命名

- ハイフン付き小文字を使用: `python-reviewer.md`
- 説明的に: `workflow.md`ではなく`tdd-workflow.md`
- エージェント/スキル名をファイル名に一致させる
